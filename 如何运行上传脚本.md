# GitHub Release 自动上传脚本使用说明

脚本位置：`upload-release.ps1`

## 快速开始

### 步骤 1：创建 GitHub Token

1. 访问：https://github.com/settings/tokens
2. 点击 **Generate new token (classic)**
3. 勾选 **repo** 权限（所有 repo 相关选项）
4. 点击 Generate token 生成
5. **复制保存** 这个 token（只显示一次！）

### 步骤 2：在 GitHub 上创建 Release

1. 访问：https://github.com/useEffectGuy/shigongrizhi/releases
2. 点击 **Draft a new release**
3. 选择 tag：`v1.0.0`
4. 填写标题和描述
5. 点击 **Publish release**（先不上传附件）

### 步骤 3：运行上传脚本

打开 PowerShell，进入项目目录：

```powershell
cd C:\Users\Administrator\Documents\trae_projects\shigongrizhi
```

设置 Token（将 `你的token` 替换为实际的 token）：

```powershell
$env:GITHUB_TOKEN = "你的token"
```

运行脚本：

```powershell
.\upload-release.ps1
```

### 成功后

脚本会显示：
```
========================================
  上传完成
========================================
  成功: 4
  跳过: 0
  失败: 0

访问 Release 页面:
  https://github.com/useEffectGuy/shigongrizhi/releases/tag/v1.0.0
```

## 常见问题

### Q: 提示"未检测到 GITHUB_TOKEN"
- 确保已设置 `$env:GITHUB_TOKEN` 环境变量
- 注意不要有多余的空格

### Q: 提示"未找到 release: v1.0.0"
- 确保已先在 GitHub 上创建了 Release
- 访问：https://github.com/useEffectGuy/shigongrizhi/releases/new

### Q: Token 权限不够
- 确保 Token 已勾选 `repo` 权限
- 重新生成 Token 并再次尝试

### Q: 想重新上传已存在的文件
- 先在 GitHub Release 页面删除该文件
- 再运行脚本
